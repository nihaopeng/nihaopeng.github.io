传统moe存在的问题

  • 知识混合

就是虽然分了不同的expert，但是输入时，有些专家分配了多个token，蕴含不同的知识

  • 知识冗余

某些输入tokens包含的相同（通用）的知识，将会被每个专家学习到，那么专家的参数就是冗余的。

解决

细粒度化expert

  • 增加expert的数量

  • 降低expert ffn的参数量

相应的，将激活的expert的数量增大到m倍，以维持计算成本近似
